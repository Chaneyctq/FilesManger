# 增量扫描逻辑修改总结：只展示新增重复组

## 1. 背景与目的
在原有的文件重复扫描功能中，每次扫描都会列出所有发现的重复文件组。当用户进行二次扫描（增量扫描）时，如果再次展示之前已经发现但用户决定保留（或未处理）的重复组，会造成信息冗余，影响用户体验。

本次修改旨在优化增量扫描体验，使用户在进行增量扫描时，**界面仅展示本次扫描新发现的重复文件组**，忽略之前已存在的重复组。

## 2. 核心逻辑修改

### 修改前
- 扫描器遍历文件系统。
- 识别出所有重复文件组。
- 无论这些重复组是否在上次扫描中出现过，全部展示给用户。

### 修改后 (只展示新增重复组)
1.  **维护历史状态**：系统需要能够识别哪些是“旧”的重复组。这通常通过对比当前的扫描结果与上一次的扫描快照（或排除已忽略/已处理的记录）来实现。
2.  **增量过滤**：
    - 在扫描完成后，对识别出的每一个重复组进行检查。
    - **判定条件**：如果一个重复组的所有成员文件路径和哈希值与上次扫描结果完全一致，则视为“旧重复组”。
    - **展示策略**：
        - **旧重复组** -> **隐藏/过滤**。
        - **新重复组**（包含新文件，或全新出现的重复组） -> **展示**。
3.  **数据更新**：虽然界面只展示新增组，但底层的重复文件数据库/缓存仍需更新，以包含所有当前的重复信息，确保后续操作的准确性。

## 3. 预期效果
- **首次扫描**：展示所有重复文件组。
- **文件无变化再次扫描**：展示结果为空（提示“未发现新增重复文件”）。
- **新增重复文件后再次扫描**：仅展示包含新加入文件的重复组，或完全由新文件组成的新重复组。

## 4. 优势
- **聚焦**：帮助用户快速定位最近产生的问题（新生成的重复文件）。
- **效率**：减少用户重复浏览已知信息的时间。
